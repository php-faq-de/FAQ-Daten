Am 30. Juni 2009 wurde die finale Version 5.3 von PHP veröffentlicht. Diese bringt, als Minor-Release, zahlreiche neue Features mit:

Einführung von Namensräumen (namespaces)

Namensräume sind ein Feature auf das die Entwicklergemeinde lange gewartet hat. Vereinfacht gesagt kann jetzt jeder vollkommen unproblematisch im eigenen Namensraum für Klassen, Funktionen und Konstanten beliebige Namen nutzen, ohne das er sich um die Kollision der Namen mit existierenden Sorgen machen muss. PHP selbst nutzt ausschließlich den Root-Namensraum "\". Mehr dazu findet sich im PHP- Manual zu Namensräumen.

Späte statische Bindung (late static binding)

Diese ermöglicht es durch Nutzung des Schlüsselworts static:: festzulegen das Vererbung auch für statische Nutzung beachtet werden soll. Genaueres dazu findet sich im PHP-Manual zu late static binding

Anonyme Funktionen (closures)

Diese ermöglichen die Nutzung von Funktionen ohne einen Namen. Das macht z.B. dann Sinn wenn eine Funktion als Callback an eine andere übergeben werden soll. Mehr dazu findet sich im PHP-Manual zu closures

Der "jump label"-Operator (goto)

Hierbei handelt es sich um einen eingeschränkten goto Befehl der es ermöglicht zu einer bestimmten Codestelle zu springen die entweder ein Label Labelname: … ist oder ein Schlüsselwort wie z.B. end in Schleifen. Details findest Du im entsprechenden PHP-Manual zu goto.

NOWDOC Schreibweise

Neben der bekannten HEREDOC Schreibweise für Zeichenketten gibt es jetzt auch die NOWDOC-Syntax. Der Unterschied zu HEREDOC ist das kein Inhalt interpretiert wird. (Also wie bei Zeichenketten die in 'Singlequotes' definiert sind) Informationen dazu findest Du im entsprechenenden Abschnitt im PHP-Manual.

Ternary-Operator in Kurzschreibweise

Der 3-fach Operator ?: existiert schon seit längeren. Neu ist das ieser jetzt auch in Kurzschreibweise genutzt werden kann. expr1 ?: expr3 gibt expr1 zurück wenn es zu TRUE evaluiert, sonst expr3. Details dazu gibt wie immer im entsprechenden PHP-Manual Abschnitt.

Dynamischer Zugriff auf statische Elemente

Man kann jetzt auch statische Methoden über eine Instanz aufrufen. Wie etwa $instance::staticMethod(). Der Vorteil hier ist das man nicht zwingend den Namen der Klasse kennen muss.

Neue Fehlermodi E_USER_DEPRECATED und E_DEPRECATED

Diese werden genutzt wenn signalisert werden soll das irgendwo veralteter Code genutzt wird.

Das ist natürlich nur ein kleiner Auszug der wichtigsten Änderungen. Wer alle Änderungen kennen will sollte einen Blick ins Changelog von PHP 5.3 werfen.
